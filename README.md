# m-app-resource

## 项目简介

`m-app-resource` 是一个用于归档和发布多个独立 HTML 页面的仓库。这些页面通常作为轻量级应用、工具页面或静态资源使用，不依赖特定域名或后端服务。本仓库旨在提供一个稳定、可访问的 HTML 资源集合，方便直接通过浏览器访问或嵌入到其他项目中。

## 目录说明

仓库目录结构如下（示例）：

```
m-app-resource/
├── pages/
│   ├── page-a/
│   │   ├── index.html
│   │   └── assets/
│   └── page-b/
│       ├── index.html
│       └── style.css
├── docs/
│   └── README.md
└── README.md
```

- `pages/`：存放独立的 HTML 页面文件夹，每个文件夹代表一个独立页面。
- `docs/`：存放额外文档或说明文件（可选）。
- 根目录下的 `README.md` 即本文件，提供仓库整体说明。

## 页面归档说明

本仓库中的每个 HTML 页面均为独立归档，不涉及后端交互或动态数据。页面内容可能包含纯前端交互、静态展示或工具功能。归档时遵循以下原则：

- 每个页面保留完整的前端资源（如 CSS、JavaScript、图片等），确保可直接在浏览器中打开使用。
- 页面命名保持语义化，便于识别用途。
- 不包含任何外部依赖或跨域资源请求，除非必要且已明确标注。

## 维护说明

本仓库由维护者定期更新，新增或修改页面时遵循以下规范：

1. **新增页面**：在 `pages/` 下创建新文件夹，放入 `index.html` 及相关资源。
2. **更新页面**：直接修改对应文件夹内容，保持历史版本可追溯（通过 Git 提交记录）。
3. **删除页面**：移除对应文件夹，并在提交说明中注明原因。
4. **资源限制**：请勿上传过大文件（如超过 10MB），避免影响仓库克隆速度。
